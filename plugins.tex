\section{Writing Plugins}
\sectionlink{writing-plugins}
\vspace{1em}

Plugins come in two flavours:

\begin{itemize}
    \item Local plugins: \mono{conftest.py} files
    \item Installable plugins via packaging entry points
\end{itemize}

\vspace{.5em}

They can contain fixtures and hook implementations for:
\vspace{.25em}

\begin{minipage}{.5\linewidth}
    \begin{itemize}
        \item configuration
        \item collection
        \item test running
        \item reporting
    \end{itemize}
\end{minipage}
\begin{minipage}{.4\linewidth}
    \includegraphics[width=7em]{img/illustrations/writingplugins.png}
\end{minipage}
\vspace{1em}

A hook is auto-discovered by its \mono{pytest_}\textcolor{codeblue}{$\cdots{}$} name.
\vspace{1em}

The arg names \textbf{must} match the precise signature names:
\vspace{-1em}

\begin{minted}[escapeinside=||, autogobble]{python}
    def pytest_configure(|\highlightbox{config}|):
        |\codeskip|
\end{minted}
\vspace{-.25em}

\begin{minipage}{.7\linewidth}
    You can accept fewer parameters:

    \begin{minted}[escapeinside=||, autogobble]{python}
    def pytest_configure():
        |\codeskip|
\end{minted}
    \vspace{-.25em}

    mismatch:

    \begin{minted}[escapeinside=||, autogobble]{python}
    def pytest_configure(|\highlightbox[highlightcolorred]{cfg}|):
        |\codeskip|
\end{minted}
\end{minipage}
\begin{minipage}{.29\linewidth}
    \includegraphics[width=5em]{img/illustrations/hooks.png}
\end{minipage}
\vspace{1em}

As with fixtures, imagine pytest calling your function with keyword arguments:

\begin{minted}[escapeinside=||, autogobble]{python}
        pytest_configure(|\highlightbox{config=}pytest.Config(\codeskip*)|)
    \end{minted}

    )
    \end{minted}\fi

\subsection{Collection: pytest\_collection\_modifyitems}
We get a list of test items and can re-order or filter them.
\vspace{-2.5em}

\begin{center}
    \begin{tikzpicture}[itemnode/.style={bluenode, align=center}]
        \node[itemnode] (func1) {Item \\ \py{def test_a():}};
        \node[itemnode, right=3mm of func1] (func2) {Item \\ \py{def test_b():}};
        \node[itemnode, right=3mm of func2] (func3) {Item \\ \py{def test_c():}};

        \draw [-, dotted] (func1) -- (func2);
        \draw [-, dotted] (func2) -- (func3);
    \end{tikzpicture}
\end{center}

\begin{minted}[autogobble,highlightlines=4]{python}
        def pytest_collection_modifyitems(
            session: pytest.Session,
            config: pytest.Config,
            items: list[pytest.Item],
        ) -> None:
\end{minted}

\emph{Called after collection has been performed. \\ May filter or re-order the items in-place.}

\vspace{1em}
We can use the hook to e.g.\ add a marker to all tests using another marker:

\begin{minted}[autogobble,highlightlines=3-4]{python}
        def pytest_collection_modifyitems(items):
            name = "skip_windows"
            for item in items:
                if item.get_closest_marker(name):
                    item.add_marker(
                        pytest.mark.skipif(
                            sys.platform == "win32",
                            reason="skipped on Windows",
                        )
                    )
\end{minted}

\subsection{Installable Plugins}
\sectionlink{plugin-installable}
\vspace{.5em}

You can package pytest like any other Python package, and add entry points to make it auto-discoverable by pytest:
\vspace{1em}

\filenameheadert{pyproject.toml}
\begin{minted}[autogobble,escapeinside=||,highlightlines={2,5-6,8-9}]{toml}
    [project]
    name = "pytest-yamlcalc"
    description = "yaml-based rpn calculator tests"
    |\codeskip|
    classifiers = ["Framework :: Pytest", |\codeskip*|]
    dependencies = ["pytest>=7.0.0", |\codeskip*|]

    [project.entry-points.pytest11]
    yamlcalc = "pytest_yamlcalc.plugin"

    [|\codeskip*|]
    |\codeskip|
\end{minted}
\vspace{2em}
